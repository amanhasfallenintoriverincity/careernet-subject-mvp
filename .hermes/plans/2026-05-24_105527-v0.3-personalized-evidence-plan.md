## Live CareerNet API 검증 메모

사용자가 제공한 CareerNet API 키로 실제 엔드포인트를 검증했다. 키 값은 저장/문서화하지 않는다.

검증 결과:

- `MAJOR` + `MAJOR_VIEW`: 구현 가능. `MAJOR` 목록에서 `majorSeq`가 오고, `MAJOR_VIEW` 상세에는 `relate_subject`가 `list` 형태로 존재한다. v0.3의 과목 근거/점수화 핵심 데이터로 사용 가능하다.
- `JOB` + `JOB_VIEW`: 구현 가능. 단, `searchJobNm=인공지능`은 결과 0건이었고 `개발자`, `소프트웨어`, `간호사` 같은 검색어는 결과가 있다. v0.3에서는 입력 키워드가 너무 구체적일 때 보조 검색어 추출/재검색을 추가하면 좋다.
- `COSE` + `COSE_VIEW`: 구현 가능하지만 현재 v0.2 mapper가 live 필드명을 일부 놓친다. 실제 응답은 제목이 `dataTitle`, 상세 본문이 `dataContent`, 첨부가 `attFile`, 대상이 `targt` 또는 `target`으로 온다. 현재 `mapMaterial()`은 `dataTitle`을 보지 않아 live 결과에서 자료가 필터링될 수 있으므로 v0.3 선행 수정이 필요하다.
- `COUNSEL` + `COUNSEL_VIEW`: 구현 가능. 목록은 `code`, `gubun`, `memo`를 제공하고 상세는 `content` 중심으로 온다. 현재 merge 구조로 질문은 목록 `memo`, 답변은 상세 `content`에서 구성 가능하다.
- `JOB_TYPE`: 구현 가능하지만 live 필드명은 `jbgp_code`, `jbgp_code_nm`이다. 현재 mapper가 이 변형을 보지 않아 결과가 빈 배열이 될 수 있으므로 v0.3 선행 수정이 필요하다.

현재 앱을 API 키와 함께 `/api/recommend?keyword=개발자`로 smoke test한 결과:

```text

majors: 3
learningMaterials: 0  # dataTitle 미매핑 때문
counselingCases: 3
jobTypes: 0           # jbgp_code/jbgp_code_nm 미매핑 때문
```

따라서 v0.3 구현 전 또는 Task 5 이전에 다음 mapper 보정 Task를 추가한다.

- `mapMaterial()` title 후보에 `dataTitle`, description 후보에 `dataContent`, url/attachment 후보에 `attFile` 계열 추가
- `mapJobType()` code 후보에 `jbgp_code`, name 후보에 `jbgp_code_nm` 추가
- `JOB` 검색 0건일 때 입력어 토큰화 또는 fallback keyword 재검색 추가 검토

## v0.3 제품 범위

### 반드시 포함할 기능

1. **학생 맞춤 입력 추가**
   - 학년: `grade=1|2|3`
   - 관심 계열: `interestArea=ai|bio|design|business|general`
   - 좋아하거나 강점 있는 과목: `preferredSubjects=정보,수학`
   - 부담스럽거나 피하고 싶은 과목: `weakSubjects=물리학Ⅰ`

2. **과목별 추천 근거 표시**
   - 예: “커리어넷 학과 상세의 관련 과목”, “입력 진로 키워드 규칙”, “학생 강점 과목과 일치”, “부담 과목으로 우선순위 낮춤”
   - UI에서 강력 추천/추가 추천 태그만 보여주는 것이 아니라, 각 과목 카드에 근거를 표시한다.

3. **점수 기반 우선순위화**
   - 현재는 `allSubjects + rule.strong + rule.optional`을 단순 병합한다.
   - v0.3에서는 subject score를 계산해 정렬한다.
   - CareerNet 근거, 키워드 규칙, 관심 계열, 학생 선호, 약점 과목을 각각 가중치로 반영한다.

4. **추천 신뢰도/데이터 커버리지 표시**
   - `CareerNet`, `Fallback` 배지만으로는 부족하므로 결과 상단에 데이터 커버리지를 표시한다.
   - 예: `직업 3건 · 학과 2건 · 자료 5건 · 상담 1건`
   - 추천 신뢰도: `높음 | 보통 | 낮음`

5. **README와 버전 갱신**
   - `package.json`, `package-lock.json`을 `0.3.0`으로 올린다.
   - README에 `v0.3 기능`과 쿼리 파라미터 예시를 추가한다.

## 점수 규칙 초안

`scoreSubjects()` 내부 기준:

| 근거 | 가중치 |
|---|---:|
| CareerNet `MAJOR_VIEW.relate_subject`에서 추출 | +5 |
| 키워드 규칙의 `strong` | +4 |
| 키워드 규칙의 `optional` | +2 |
| 관심 계열 기본 과목과 일치 | +3 |
| 학생 선호/강점 과목과 일치 | +3 |
| 학생 부담 과목과 일치 | -4 |

정렬:

1. `score` 내림차순
2. CareerNet 근거가 있는 과목 우선
3. 기존 출현 순서 유지

티어:

- `strong`: score >= 7 또는 상위 4개
- `optional`: score >= 3
- `explore`: 나머지

```ts
export type StudentProfile = {
  grade?: '1' | '2' | '3';
  interestArea?: 'ai' | 'bio' | 'design' | 'business' | 'general';
  preferredSubjects?: string[];
  weakSubjects?: string[];
};

const GRADES = new Set(['1', '2', '3']);
const INTEREST_AREAS = new Set(['ai', 'bio', 'design', 'business', 'general']);

type QueryValue = string | string[] | undefined;

type QueryLike = Record<string, QueryValue>;

function first(value: QueryValue): string | undefined {
  return Array.isArray(value) ? value[0] : value;
}

function parseSubjectList(value: QueryValue): string[] | undefined {
  const raw = Array.isArray(value) ? value.join(',') : value;
  const items = Array.from(new Set((raw ?? '').split(/[,.，、|/\n]+/).map((item) => item.trim()).filter(Boolean)));
  return items.length ? items : undefined;
}

export function serializeSubjectList(subjects: string[]): string {
  return Array.from(new Set(subjects.map((item) => item.trim()).filter(Boolean))).join(',');
}

export function parseStudentProfile(query: QueryLike): StudentProfile {
  const grade = first(query.grade);
  const interestArea = first(query.interestArea);
  const profile: StudentProfile = {};

  if (grade && GRADES.has(grade)) profile.grade = grade as StudentProfile['grade'];
  if (interestArea && INTEREST_AREAS.has(interestArea)) profile.interestArea = interestArea as StudentProfile['interestArea'];

  const preferredSubjects = parseSubjectList(query.preferredSubjects);
  const weakSubjects = parseSubjectList(query.weakSubjects);
  if (preferredSubjects) profile.preferredSubjects = preferredSubjects;
  if (weakSubjects) profile.weakSubjects = weakSubjects;

  return profile;
}
```

function buildDataCoverage(input: RecommendationInput): DataCoverage {

- 기존 `strong`, `optional`, `reason` 필드는 유지한다.
- 새 필드 `scored`, `dataCoverage`, `studentProfile`은 optional 또는 추가 필드로 둔다.

### 4. 서버 컴포넌트 폼이 복잡해질 수 있음

대응:

- 클라이언트 상태 관리 없이 GET form만 사용한다.
- 입력값 유지도 `defaultValue`와 URL 쿼리로 해결한다.
